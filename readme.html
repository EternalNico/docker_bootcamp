Documentation du Projet Docker

Résumé des exercices et fonctionnalités implémentées

Exercice 1 : Application Flask

  • Déploiement d'une application Flask simple.
  • Exposition du port 5000 pour accéder à l'application.
  • Utilisation d'un utilisateur non-root pour des raisons de sécurité.

Exercice 2 : Multi-stage Build

  • Utilisation de Node.js pour construire des assets statiques.
  • Déploiement des fichiers statiques avec Nginx.
  • Optimisation de l'image Docker grâce à un multi-stage build.

Exercice 3 : Gestion des Volumes

  • Configuration de volumes pour la persistance des données et des logs.
  • Montage en lecture seule pour les fichiers de configuration.

Exercice 4 : Reverse Proxy avec Nginx

  • Configuration de Nginx comme reverse proxy pour une application web.
  • Exposition des ports pour accéder à l'application via Nginx.

Exercice 5 : Application Flask avec Nginx

  • Déploiement d'une application Flask derrière un reverse proxy Nginx.
  • Configuration des dépendances entre les services.

Exercice 6 : Gestion des Environnements

  • Utilisation de fichiers `.env` pour gérer les variables d'environnement.
  • Configuration distincte pour les environnements de développement et de production.

Exercice 7 : Scale et Load Balancing

  • Scale horizontal des services Flask avec plusieurs instances.
  • Configuration de Nginx pour le load balancing.
  • Ajout de Prometheus et Grafana pour le monitoring.

Exercice 8 : Healthchecks

  • Implémentation de routes `/health` pour vérifier la disponibilité des services.
  • Configuration de Docker pour redémarrer automatiquement les conteneurs en cas de défaillance.

Exercice 9 : Sécurité

  • Utilisation d'utilisateurs non-root pour les conteneurs.
  • Limitation des ressources CPU et mémoire pour chaque service.
  • Isolation des services dans des réseaux Docker personnalisés.

Exercice 10 : Déploiement

  • Gestion des secrets avec Docker Secrets.
  • Configuration de volumes persistants pour les données.
  • Stratégies de mise à jour des services avec Docker Compose.

Comment lancer le projet

Voici les étapes pour lancer le projet :

  1. Assurez-vous que Docker et Docker Compose sont installés sur votre machine.
  2. Clonez le dépôt ou placez-vous dans le répertoire du projet.
  3. Construisez et démarrez les services avec la commande suivante :
    4. docker-compose up --build
  4. Pour arrêter les services, utilisez la commande :
    5. docker-compose down

Accéder aux services

Une fois les services démarrés, vous pouvez y accéder via les ports configurés :

  • Application Flask : http://localhost:10017
  • Nginx (reverse proxy) : http://localhost:10018
  • Prometheus (monitoring) : http://localhost:9090
  • Grafana (visualisation) : http://localhost:3000

Commandes utiles

Voici quelques commandes utiles pour interagir avec les conteneurs :

  • Voir les conteneurs en cours d'exécution :
    • docker ps
  • Voir les logs d'un conteneur :
    • docker logs [nom_du_conteneur]
  • Se connecter à un conteneur :
    • docker exec -it [nom_du_conteneur] sh
  • Supprimer tous les conteneurs et volumes :
    • docker-compose down --volumes
